## 3. Neural ODE for Parameter Estimation (`neural_ode`)

This implementation treats physical parameter estimation as a **continuous-time trajectory optimization** problem. Following the notation in the root README, the model dynamics are defined by an Ordinary Differential Equation (ODE) where the parameters are learned via gradient-based optimization.

### How it Works:
* **Continuous Transition**: The system state evolves according to a prescribed physical law $f$, where the physical parameters $\theta$ (e.g., $\sigma, \rho, \beta$ for Lorenz-63) are treated as learnable weights:
    $$\frac{dx}{dt} = f(x, t, \theta)$$
* **Trajectory Integration**: Given an initial state $x_0$ and a set of parameters $\theta$, the entire predicted trajectory $\hat{x}_{1:T}$ is generated by an ODE solver over the observation interval:
    $$\hat{x}_{1:T} = \text{ODESolve}(x_0, f, \theta, t_{1:T})$$
* **Objective Function**: A global loss function $L(\theta)$ is defined to measure the discrepancy between the integrated trajectory and the discrete observations $y_k$:
    $$L(\theta) = \sum_{k=1}^T \|y_k - h(\hat{x}_k(\theta))\|^2$$
* **Gradient via Adjoint Method**: To update $\theta$ without the memory cost of storing intermediate solver states, the gradient $\nabla_\theta L(\theta)$ is computed by solving an adjoint ODE backward in time:
    $$\frac{da}{dt} = -a(t)^T \frac{\partial f}{\partial x}$$
    
* **Gradient Descent**: The parameters are then updated using an optimizer (e.g., Adam or L-BFGS):
    $$\theta \leftarrow \theta - \alpha \nabla_\theta L(\theta)$$

---

### Comparison to EnKF Methods:
* **Optimization Style**: Unlike the **State Augmented EnKF**, which updates parameters sequentially at each timestep, the Neural ODE performs a batch update over the entire time horizon.
* **Differentiability**: Like the **Autodiff EnKF**, this method relies on backpropagating through the model. However, it differentiates through the continuous ODE solver rather than the discrete Kalman filtering steps.
* **Data Usage**: This approach is highly effective for irregularly sampled data or long-term trajectory reconstruction, as the solver treats time as a continuous variable.
